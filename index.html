Foodies Milestones

Early Ideation

In the early ideation phase, I took the lead in proposing the initial concept and features for the Foodies project. This involved brainstorming ideas with the team, laying out the platform's core objectives, and identifying features that would increase user engagement. My proposed features included an interactive map for food truck locations, a calendar for upcoming events, and a booking system for private events.

This sketch (see Image 5) showcases the initial wireframe I designed for the Foodies platform.

Along with the feature set, I also created the initial wireframe, which served as the foundation for the platform's design and functionality.

This sketch (see Image 6) showcases the initial wireframe I designed for the Foodies platform.

Research & Inspiration

During the research phase, I analyzed several projects that aligned with the goals of the Foodies platform. One key source of inspiration was the Brisbane City Council Food Truck website, which provides real-time updates on food truck locations across the city. This inspired our integration of real-time data and an interactive map to elevate user engagement.

Map page of Brisbane food truck (See Image 7). Reference: Brisbane City Council Food Truck, https://www.bnefoodtrucks.com.au/

Prototyping

In the prototyping stage, I took on the primary development role, creating the base mockup pages and setting up seamless page navigation using JavaScript. After we completed the mid-fidelity prototype, we gathered user feedback that helped us refine key design elements. These adjustments shaped the final prototype, reflecting our MVP features \ and enhancing the overall user experience.

This mid-fidelity prototype (see Image 8) represents the second stage of our design, incorporating feedback from early testing.

In addition, I developed the backend using PHP, integrating external APIs for both the Food Truck and Market data. I was responsible for building the core pages, including the index(map), and details pages, which dynamically display real-time content fetched from the APIs.

During prototype user testing, we received some useful feedbacks (See Image 9).

This image (see Image 10) from my GitHub commit history showcases the early stages of development, where I implemented the backend and API fetching capabilities.

Our final prototype website (see Image 11) highlights our MVP features, such as the integrated food truck and market API, which formed the backbone of our real-time data functionality.

Refinement

During the refinement phase, I worked on improving the user experience and flow based on team feedback and testing results. I implemented the landing screen (see Image 12) to showcase the platform's features and made final adjustments to ensure the functionality aligned with the MVP goals.

Reflecting on the user feedback that the top navigation bar was difficult to use, I implemented a collapsible menu to improve usability (see Image 13). During this phase, I took on the responsibility of assigning tasks to the team and managing the overall project guidelines, ensuring everything stayed on track. This was particularly important as there had been earlier confusion about roles, and I wanted to make sure each team member had clear direction moving forward.

After multiple iterations and improvements, we are closer to the final refined version of the website, which offers an optimized user experience and highlights our core features.
